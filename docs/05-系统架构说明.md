## 数据流详情

```
Step 1: CSV文件                     →  原始数据 (~4,700行)
Step 2: Spark ComplaintDataClean    →  清洗数据 (4,670行, ORC格式)
Step 3: Spark BrandAnalysis         →  品牌统计 (118品牌)
Step 4: Spark SeriesAnalysis        →  车系统计 (670车系)
Step 5: Spark ProblemCategoryAnalysi→  问题大类 (19类)
Step 6: Spark ProblemTypeAnalysis   →  问题类型 (110种)
Step 7: Spark BrandProblemCross     →  交叉关联 (733条)
Step 8: Spark DailyTrendAnalysis    →  日趋势 (92天)
Step 9: LOAD DATA LOCAL INFILE      →  MariaDB 6表
Step 10: Spring Boot JdbcTemplate   →  JSON API
Step 11: Fetch API + ECharts        →  可视化大屏
```

---

## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 清洗输出格式 | ORC | 列式存储，压缩率高，后续分析高效 |
| 分析输出格式 | CSV | 纯文本，MariaDB LOAD DATA直接导入 |
| Spark运行模式 | local[2] | 小数据量(4700条)，绕过YARN调度延迟 |
| 后端框架 | Spring Boot 2.7 | Java 8兼容，内嵌Tomcat，自动配置 |
| 数据库连接 | JdbcTemplate | 6个简单查询，无需ORM |
| 字符集 | utf8mb4 | 支持完整Unicode，避免生僻字问题 |
| 前端 | 原生JS+ECharts CDN | 零构建依赖，单HTML文件快速部署 |

## 性能数据

| 阶段 | 耗时 | 说明 |
|------|------|------|
| Spark清洗 | ~8s | local[2]模式，4700条数据 |
| 6维分析 | ~15s | 共6次GROUP BY查询 |
| LOAD DATA导入 | <1s | 6表共1742行 |
| API响应 | ~30ms | 千级数据，无索引 |
| 页面首屏 | ~500ms | 6个并发Fetch |
